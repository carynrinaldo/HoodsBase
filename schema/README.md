# schema/ ‚Äî Generated Schema Files

This directory contains the auto-generated database schema and the scripts that produce it. Nothing here is hand-edited except the generator scripts themselves.

## Files

| File | Purpose |
|------|---------|
| `mappings.yml` | Auto-generated API field structure, pagination info, child resource annotations. Source of truth for what the API returns. Generated by `system/rebuild_mappings.py`. |
| `context.yml` | Auto-generated business context ‚Äî field renames, type overrides, flatten/skip rules. Merged from `mappings.yml` + `system/api_knowledge.yml` by `system/rebuild_context.py`. |
| `generate_schema.py` | Reads `mappings.yml` + `context.yml` ‚Üí writes `schema.sql` |
| `generate_views.py` | Reads `context.yml` ‚Üí creates `v_` views with `_dt` timestamp columns in the live database |
| `schema.sql` | Generated SQLite DDL (~3,300 tokens). Doubles as Claude's MCP system prompt. |

## Generators

### generate_schema.py

Produces `schema.sql` from the two YAML files. The output is deliberately kept as clean DDL ‚Äî no pragmas, no `IF NOT EXISTS`, no runtime settings ‚Äî because it doubles as Claude's MCP system prompt. Inline comments are limited to enums and non-obvious semantics.

```bash
docker exec safehoods-dev python /app/schema/generate_schema.py
```

### generate_views.py

Creates `v_` views in the live database for every table that has timestamp fields. Each view selects all base columns plus `datetime(col, 'unixepoch', 'localtime') AS col_dt` for each timestamp column. Idempotent ‚Äî uses `DROP VIEW IF EXISTS` before each `CREATE VIEW`.

```bash
docker exec safehoods-dev python /app/schema/generate_views.py
```

**Views created (12):** `v_company`, `v_location`, `v_contact`, `v_job`, `v_appointment`, `v_invoice`, `v_invoice_item`, `v_quote`, `v_asset`, `v_deficiency`, `v_service_recurrence`, `v_user`

**Not created (no timestamps):** service_line, tag, payment_terms, quote_item

Query example:
```sql
SELECT name, status, scheduled_date_dt, completed_on_dt
FROM v_job WHERE status = 'completed' LIMIT 5;
```

## Pipeline

Both generators are called automatically by `system/rebuild_all.py`:

```
system/rebuild_mappings.py  ‚Üí  mappings.yml       (step 1, live API)
system/rebuild_context.py   ‚Üí  context.yml         (step 2)
generate_schema.py          ‚Üí  schema.sql          (step 3)
generate_views.py           ‚Üí  v_ views in DB      (step 4, skipped if DB doesn't exist)
```

Run the full pipeline: `python system/rebuild_all.py` (or `--skip-mappings` to skip API calls).

## Design Decisions

- **schema.sql is token-optimized** for use as an MCP system prompt. Conventions (`_id` = FK, `_dt` = timestamp companion, INTEGER booleans, JSON TEXT columns) are documented once in a header rather than repeated per column.
- **Views live in the database, not in schema.sql.** This keeps the MCP prompt lean while giving Claude and the user human-readable dates via `v_` views.
- **`_dt` companion columns** are computed at query time by SQLite's `datetime()` function using the container's local timezone (America/Los_Angeles). They are not stored ‚Äî the base tables keep only the raw Unix epoch integers.
